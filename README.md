# Договор комиссии — браузерная утилита для автосалона

Одностраничное веб‑приложение: сотрудник загружает фотографии документов клиента → ИИ
извлекает данные → сотрудник проверяет и правит → получает готовый к печати
договор комиссии в формате `.docx` по шаблону салона.

Язык интерфейса — русский. Регион — Беларусь.

## Возможности

- **Мастер из 4 шагов**: Загрузка → Проверка → Реквизиты → Документы.
  1. **Загрузка** — 5 фотографий: три страницы паспорта (фото/MRZ, разворот с ФИО,
     страница с пропиской) + обе стороны тех. паспорта (лицевая — характеристики,
     оборотная — владелец и адрес). Каждая страница распознаётся отдельным
     фокусированным запросом к **OpenRouter** по своей схеме.
  2. **Проверка** — внутренняя навигация по блокам: слева фотография, справа
     распознанные поля для сверки и правки (паспорт-фото, ФИО+орган, тех. паспорт,
     адрес регистрации с двумя фото).
  3. **Реквизиты** — ручной ввод (№ и дата договора, пробег, суммы).
  4. **Документы** — предпросмотр и скачивание **двух документов**: договора
     комиссии и акта приёма-передачи (.docx / .pdf).
- **Сверка адреса прописки**: адрес берётся из паспорта (источник истины); если он
  отличается от адреса на тех. паспорте — поле подсвечивается для проверки.
- Загрузка с диска или **камеры телефона**.
- Учёт двуязычности паспорта РБ (берётся РУССКИЙ вариант), нормализация регистра
  (ФИО — в обычном регистре, СРТС — заглавными), орган выдачи — в творительном падеже,
  суммы прописью со склонением валюты.
- **Сохранение прогресса**: данные и фотографии хранятся в браузере (localStorage),
  переживают переходы вперёд/назад и перезагрузку страницы. Сбрасываются только
  кнопкой «Новый договор».
- Поля редактируемы; ненадёжные и невалидные (VIN ≠ 17 символов) подсвечены
  подсказкой «проверьте вручную».
- Ручной ввод полей договора: № и дата договора, пробег, стоимость и вознаграждение
  (сумма прописью подставляется автоматически и редактируется).
- **Точный предпросмотр**: в браузере рендерится реальный `.docx` (docx-preview),
  split view на десктопе.
- Скачивание готового документа в **`.docx`** и **`.pdf`** (PDF — через LibreOffice,
  для пиксель‑точной печати независимо от версии Word).
- Грамматика: ФИО и текст в обычном регистре, СРТС — заглавными; орган выдачи в
  творительном падеже («Выдан Слуцким РОВД…»); суммы прописью со склонением валюты
  («…три доллара США», «триста белорусских рублей 00 копеек»). Дата договора — сегодня.

## Технологии

- **Frontend:** React 18 + Vite + TailwindCSS (свои компоненты, без UI‑китов).
- **Backend:** Node.js + Express.
- **Загрузка:** multer `memoryStorage` — файлы только в памяти, на диск не пишутся.
- **Сжатие:** sharp (даунскейл до ~1500px по длинной стороне).
- **Документ:** docxtemplater + pizzip.
- **HTTP к модели:** нативный `fetch`.

## Установка и запуск

Требуется Node.js 18+ (рекомендуется 20/22+).

```bash
npm install
cp .env.example .env      # и впишите свой OPENROUTER_API_KEY
```

### Разработка

```bash
npm run dev
```

- Vite dev‑сервер: http://localhost:5173 (открывайте его).
- Express‑бэкенд: http://localhost:3000 (Vite проксирует на него `/api`).

### Прод

```bash
npm run build     # собирает фронт в dist/
npm start         # Express отдаёт dist/ и API на порту PORT
```

## Переменные окружения

| Переменная           | Описание                                                          |
|----------------------|-------------------------------------------------------------------|
| `OPENROUTER_API_KEY` | ключ OpenRouter (обязательно)                                     |
| `OPENROUTER_MODEL`   | модель распознавания, по умолчанию `qwen/qwen3.5-397b-a17b`. Альтернативы: `google/gemini-3-pro-preview` (лучший OCR/рукопись), `google/gemini-3-flash-preview`, `anthropic/claude-sonnet-4.6` |
| `AUTH_PASSWORD`      | пароль для входа. Если не задан (и нет `AUTH_PASSWORDS`) — вход отключён |
| `AUTH_PASSWORDS`     | несколько паролей через запятую (напр. для владельца и клиента)   |
| `AUTH_SECRET`        | секрет для подписи сессии (длинная случайная строка). Если не задан — выводится из пароля. Сессия живёт 90 дней |
| `PORT`               | порт бэкенда, по умолчанию `3000`                                 |

## Безопасность и данные

- **Никакой записи на диск.** Изображения держатся только в буфере памяти (multer
  `memoryStorage`) и обнуляются сразу после использования. Готовый `.docx` отдаётся
  из памяти, не сохраняется.
- **Нет базы данных и постоянного хранилища.** Состояние живёт на клиенте на время сессии.
- **Не логируется** содержимое запросов, изображения и персональные данные — только
  технические события (метод, путь, статус, длительность, текст ошибки без PII).
- В каждом запросе к OpenRouter передаётся `provider: { data_collection: "deny" }`.

### Требования к деплою

- **HTTPS/TLS обязателен.** Приложение рассчитано на работу за реверс‑прокси
  (nginx/Caddy/Traefik), который терминирует TLS. Не публикуйте HTTP наружу.
- **Отключите логирование в аккаунте OpenRouter.** В настройках OpenRouter
  (Privacy / Data policy) выключите хранение запросов и ответов. `data_collection: "deny"`
  в запросе — необходимое, но не достаточное условие: настройка аккаунта первична.
- Лимит загрузки — 10 МБ на файл, принимаются только изображения (проверка MIME).
- **PDF‑экспорт требует LibreOffice** на хосте (`soffice` в PATH или путь в
  `LIBREOFFICE_PATH`). Установка: macOS — `brew install --cask libreoffice`,
  Debian/Ubuntu — `apt-get install libreoffice`. Конвертация использует временный
  каталог в системном tmp, который удаляется сразу после чтения PDF (на диске
  ничего не остаётся). Без LibreOffice кнопка PDF вернёт понятную ошибку, а
  скачивание `.docx` продолжит работать.

## Два режима

После входа оператор выбирает тип сделки (экран «Что оформляем?»):

- **Комиссия** — клиент сдаёт авто на реализацию. По 5 фото (паспорт + тех. паспорт) →
  **договор комиссии** и **акт приёма-передачи** (`shared/contractData.js`).
- **Продажа (ДКП)** — пришёл покупатель на комиссионное авто. Оператор загружает
  **ранее сгенерированный договор комиссии (.docx)** и 3 фото паспорта покупателя →
  **договор купли-продажи** и **акт приёма-передачи к нему** (`shared/saleData.js`).
  - Характеристики ТС и № / дата договора комиссии берутся из загруженного .docx
    (`server/services/parseCommission.js`, маршрут `POST /api/parse-commission`).
  - ФИО, паспорт и прописка покупателя — из фото (тот же конвейер OpenRouter).
  - № и дата ДКП (общие для ДКП и акта, поддерживается «Б/Н») и **стоимость ТС
    в рублях прописью** — вводятся вручную.

Шаги одинаковые для обоих режимов: Загрузка → Проверка → Реквизиты → Документы.

## Шаблоны документов

В корне проекта четыре шаблона:
- `template.docx` — договор комиссии;
- `template_akt.docx` — акт приёма-передачи (комиссия);
- `template_dkp.docx` — договор купли-продажи (продажа);
- `template_akt_dkp.docx` — акт приёма-передачи к ДКП (продажа).

Договор комиссии и акт к нему заполняются **одним набором данных** (`shared/contractData.js`);
ДКП и акт к нему — другим (`shared/saleData.js`). Акт в каждом случае привязан к номеру и
дате своего договора, реквизиты и подписи те же.

Плейсхолдеры шаблонов продажи (`{...}`): `{contract_number}` `{contract_date}` `{comm_number}`
`{comm_date}` `{fio}` `{fio_short}` `{identification_number}` `{passport_number}` `{issued}`
`{reg_address}` `{vehicle_type}` `{body_type}` `{make_model}` `{vin}` `{color}` `{mileage}`
`{year}` `{reg_number}` `{cert_line}` `{price}` `{price_words}`. Здесь `{comm_number}`/`{comm_date}` —
номер и дата договора комиссии (ссылка в п. 1.2 ДКП), `{cert_line}` — «№ … от …» свидетельства.

## Как заменить шаблон

Шаблоны — обычные `.docx` в корне проекта. Чтобы использовать свой:

1. Возьмите свой `.docx`.
2. В местах, которые должны подставляться, впишите плейсхолдеры в фигурных скобках,
   например `{fio}`, `{vin}`, `{price}` (см. полный список ниже). Важно, чтобы каждый
   плейсхолдер был набран одним «куском» текста (без смены форматирования внутри скобок),
   иначе Word может разбить его на части.
3. Замените `template.docx` в корне. Перезапуск не требуется в dev (шаблон читается
   при первом рендере); в проде перезапустите процесс.

Доступные плейсхолдеры:

`{contract_number}` `{contract_date}` `{fio}` `{fio_short}` `{identification_number}`
`{passport_number}` `{issuing_authority}` `{issue_date}` `{reg_address}` `{vehicle_type}`
`{body_type}` `{make_model}` `{vin}` `{color}` `{mileage}` `{year}` `{reg_number}`
`{certificate_number}` `{price}` `{price_words}` `{commission}` `{commission_words}`

`{reg_address}` — адрес регистрации (прописка) из нижнего штампа; `{fio_short}` —
«Фамилия И.О.» для строки подписи.

> Текущий `template.docx` сгенерирован из присланного образца договора: места, выделенные
> в исходнике красным, заменены на соответствующие плейсхолдеры с сохранением вёрстки.
